# tachi-lens v0.2.0-beta.1

發佈日期：2026-07-15  
類型：Pre-release

這是 `v0.1.0` 之後的第一個測試版本，集中交付 Gemini quota-aware 排程、DeepSeek overflow，以及相關的錯誤分類與重試修正。

## 主要更新

### Quota-aware Gemini 排程

- 依 Gemini 模型分離 RPM、TPM 與 RPD quota 狀態。
- 使用持久化 reservation，避免 Service Worker 重啟後重複計費或遺失額度狀態。
- 分離 monotonic clock 與 wall clock，讓 cooldown、deadline 與 provider-day 邊界使用正確的時間來源。
- 支援 exactly-once reservation settlement、deadline 與 provider cooldown。

### DeepSeek overflow 與錯誤處理

- Gemini quota 不足、進入 cooldown 或暫時不可用時，可將工作降級至 DeepSeek。
- 保留直接選擇 DeepSeek 時的 provider、HTTP status、authentication、bad request 與 rate-limit 分類。
- Gemini quota fallback 仍保留可重試的 Gemini 429 語意與 retry timing。

### Live / backlog 公平性

- 即時聊天室訊息仍維持優先。
- backlog 存在時，連續處理 live 工作的數量有上限，避免 backlog 永久飢餓。
- 各 priority lane 內維持 FIFO。

### 診斷與可靠性

- Popup 顯示更具體的 quota 與 fallback 診斷。
- Diagnostics 不保存聊天室原文、username 或 API key。
- 加強 cache ordering、in-flight coalescing、provider isolation 與 regression coverage。

## 已知限制

- 這是 beta 版本，正式 `v0.2.0` 前仍需完成真實 Chrome / Twitch smoke test。
- GitHub CI 目前尚未執行 packaged-extension Playwright E2E；相關基礎架構由 #75 追蹤。
- 無可信 reservation timestamp 的舊版 quota snapshot 會 fail closed，直到 storage 被修復；追蹤於 #63。
- Service Worker 重啟後首次觀察到的 wall-clock forward jump，無法與合法離線時間完全區分；追蹤於 #64。

## 安裝

1. 從 GitHub Release 下載 `tachi-lens.zip`。
2. 解壓縮檔案。
3. 開啟 `chrome://extensions`。
4. 啟用「開發人員模式」。
5. 選擇「載入未封裝項目」，載入包含 `manifest.json` 的資料夾。

## 升級注意事項

- API key 與一般設定仍保存在本機 `chrome.storage`。
- 本版本新增 quota persistence 狀態。由舊版升級後，若偵測到不可信的舊 snapshot，Gemini quota path 可能暫時 fail closed，但 DeepSeek overflow 仍可使用。
